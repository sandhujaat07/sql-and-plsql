PL/SQL Block Structure

PL/SQL code is organized into blocks, which can be nested within each other. A basic block structure consists of three sections:

  • Declaration Section
  • Executable Section
  • Exception Handling Section

The Declaration Section is where variables, constants, cursors, and other PL/SQL constructs are declared. It is optional, but if used, it must be at the beginning of the block.

The Executable Section contains the actual PL/SQL code to be executed. It follows the Declaration Section (if present) and consists of one or more PL/SQL statements.

The Exception Handling Section is used to handle exceptions that may occur during the execution of the block. It is optional, but if used, it must be at the end of the block.

Example:

            
DECLARE
    -- Declaration Section
    v_variable NUMBER := 10;
BEGIN
    -- Executable Section
    IF v_variable > 0 THEN
        DBMS_OUTPUT.PUT_LINE('Variable is positive');
    ELSE
        DBMS_OUTPUT.PUT_LINE('Variable is non-positive');
    END IF;
EXCEPTION
    -- Exception Handling Section
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('An error occurred: ' || SQLERRM);
END;

In this example, we have a PL/SQL block that declares a variable (v_variable) in the Declaration Section, executes a simple conditional statement in the Executable Section to check if the variable is positive or not, and includes exception handling in the Exception Handling Section to handle any errors that may occur during execution.

PL/SQL Data Types, Variables, and Constants

PL/SQL supports various data types such as VARCHAR2, NUMBER, DATE, BOOLEAN, etc. Variables are used to store data temporarily, and constants are used to store fixed values. Here's an example: 

            
DECLARE
    v_name VARCHAR2(50) := 'John';
    v_age NUMBER := 30;
    v_is_active BOOLEAN := TRUE;
    v_pi CONSTANT NUMBER := 3.14;
BEGIN
    -- PL/SQL statements
END;

In this example, v_name is a variable of type VARCHAR2, v_age is a variable of type NUMBER, v_is_active is a variable of type BOOLEAN, and v_pi is a constant.

Scope and Visibility of a Variable

The scope of a variable determines where in the code it can be referenced, and visibility determines whether it can be accessed from other parts of the code. Here's an example: 

            
DECLARE
    v_outer_variable NUMBER := 10;
BEGIN
    DECLARE
        v_inner_variable NUMBER := 20;
    BEGIN
        DBMS_OUTPUT.PUT_LINE('Inner variable: ' || v_inner_variable);
    END;
    DBMS_OUTPUT.PUT_LINE('Outer variable: ' || v_outer_variable);
END;

In this example, v_inner_variable is declared within the inner block and is only visible within that block, while v_outer_variable is declared in the outer block and is visible throughout the entire block.

Assignments and Expressions

Assignments in PL/SQL are done using the assignment operator (:=), and expressions are constructed using variables, constants, and operators. Here's an example: 

            
DECLARE
    v_num1 NUMBER := 10;
    v_num2 NUMBER := 5;
    v_result NUMBER;
BEGIN
    v_result := v_num1 + v_num2;
    DBMS_OUTPUT.PUT_LINE('Result: ' || v_result);
END;

In this example, v_result is assigned the value of v_num1 + v_num2, and the result is printed using DBMS_OUTPUT.PUT_LINE.

Conditional and Iterative Control

PL/SQL provides control structures like IF-THEN-ELSE for conditional execution and loops like FOR and WHILE for iterative execution. Here's an example: 

            
DECLARE
    v_num NUMBER := 10;
BEGIN
    IF v_num > 0 THEN
        DBMS_OUTPUT.PUT_LINE('Number is positive');
    ELSE
        DBMS_OUTPUT.PUT_LINE('Number is non-positive');
    END IF;

    FOR i IN 1..5 LOOP
        DBMS_OUTPUT.PUT_LINE('Iteration: ' || i);
    END LOOP;
END;

In this example, the IF-THEN-ELSE statement checks if v_num is positive or not, and the FOR loop prints numbers from 1 to 5.

Cursor Management in PL/SQL

Cursors in PL/SQL are used to retrieve data from the database. They can be either explicit or implicit. Here's an example of an explicit cursor: 

            
DECLARE
    CURSOR c_employee IS
        SELECT * FROM employees;
    v_emp employees%ROWTYPE;
BEGIN
    OPEN c_employee;
    LOOP
        FETCH c_employee INTO v_emp;
        EXIT WHEN c_employee%NOTFOUND;
        -- Process the fetched record
        DBMS_OUTPUT.PUT_LINE('Employee Name: ' || v_emp.first_name || ' ' || v_emp.last_name);
    END LOOP;
    CLOSE c_employee;
END;

In this example, an explicit cursor c_employee is declared to fetch records from the employees table, and then the records are processed within a loop.

Exception Handling in PL/SQL

Exception handling in PL/SQL allows you to handle errors gracefully. You can use the EXCEPTION section to catch and handle specific exceptions. Here's an example: 

            
DECLARE
    v_num1 NUMBER := 10;
    v_num2 NUMBER := 0;
    v_result NUMBER;
BEGIN
    v_result := v_num1 / v_num2;
    DBMS_OUTPUT.PUT_LINE('Result: ' || v_result);
EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('Error: Division by zero');
END;

In this example, a division by zero error is caught and handled using the ZERO_DIVIDE exception.
